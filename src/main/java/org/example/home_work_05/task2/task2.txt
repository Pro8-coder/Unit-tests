Задание 2. *Ниже список тестовых сценариев. Ваша задача - определить тип каждого
теста (юнит-тест, интеграционный тест, сквозной тест) и объяснить, почему вы так
решили.

1) Проверка того, что функция addContact корректно добавляет новый контакт в список
контактов.
Проверка того, что функция addContact корректно добавляет новый контакт в список
контактов - это юнит-тест. Этот тест сосредоточен на проверке отдельной функции
(addContact) без участия других компонентов системы.

2) Проверка того, что при добавлении контакта через пользовательский интерфейс,
контакт корректно отображается в списке контактов.
Проверка того, что при добавлении контакта через пользовательский интерфейс,
контакт корректно отображается в списке контактов является примером интеграционного
теста. Этот тест охватывает взаимодействие между пользовательским интерфейсом и
списком контактов, что делает его типичным для интеграционных тестов.

3) Проверка полного цикла работы с контактом: создание контакта, его редактирование
и последующее удаление.
Проверка полного цикла работы с контактом: создание контакта, его редактирование и
последующее удаление - это сквозной тест. Этот тест проверяет взаимодействие между
различными компонентами системы (создание, редактирование, удаление контакта) и
оценивает полный жизненный цикл объекта в системе.
